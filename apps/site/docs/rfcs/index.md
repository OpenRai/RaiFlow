# RFCs

RaiFlow uses short RFCs to document architecture decisions that are meant to survive implementation churn.

Important caveat:

RFCs describe the intended v2 system design. The codebase is still catching up to parts of that design. For current implementation status, read the roadmap as well.

## Current RFCs

- [`0001`](./0001-project-framing) — project framing and scope fence
- [`0002`](./0002-observe-mode) — unified runtime architecture
- [`0003`](./0003-event-model) — resource and event model

## What RFCs Are For

RFCs exist to:

- define terms
- freeze major architectural decisions
- record tradeoffs
- explain why a direction was chosen

They are not release notes and they are not proof that every described surface is already implemented.
